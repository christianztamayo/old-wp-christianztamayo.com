Simplify our selectors

  • Never be too specific in using selectors
  • Simplifying selectors will not only clean up our code, but will also yield better performance
  • Inheritance will be easier to override

Example, we wanted to select .sidelinks li on the first .inner-div:


<div class="container page-id-4">
  <div class="outer-div">
    <div class="inner-div first">
      <div class="wrap">

            <li>Fugit ullam sit excepturi ipsum repellat.</li>
          </ul>
        </div>

    </div><!-- This is from the first .inner-div-->
    <div class="inner-div second">
      <div class="wrap">
        <div class="sidelinks">

            <li>Fugit ullam sit excepturi ipsum repellat.</li>

Instead of using...


<style>
    div.container.page-id-4 .outer-div .inner-div.first .wrap .sidelinks li {
        /* style here */
    }
</style>

We can use...


<style>
    .page-id-4 .inner-div.first .sidelinks li {
        /* style here */
    }
</style>

Or much further, depending on the broadness of the class...


<style>
    .inner-div.first .sidelinks li {}
    .inner-div:first-child .sidelinks li {}
</style>

We can also use :first-child. That way, we can remove the extra .first HTML class on the inner-div.

Avoid using IDs as much as possible

  1. IDs are OK, but it defeats the purpose of reusability
  2. Elements are not re-usable on that page
  3. Should only be used on special occassions
  4. Hard to override its styling
  5. IDs can lead to the "!important" bloat

Example, we want to override #dark


<style>
    #dark { background: black; }

    /* Other styles here */
    /* Other styles here */
    /* Other styles here */
    /* Other styles here */

    /* Then came a time that we wanted to override #dark */
    .notdark { background: yellow; }

</style>
<p id="dark" class="notdark">test</p>

Simplify Media Queries

In our case, the Genesis Framework, we have three breakpoints to consider. These are:

  • @media only screen and (max-width: 1139px) {}
  • @media only screen and (max-width: 1023px) {}
  • @media only screen and (max-width: 767px) {}

We lock ourselves into these three. As much as possible, we don't want to add more breakpoints to fix things. Instead, we can just adjust our styling, or even rewrite our code. Let's make sure we think it through before writing any code.

Wordpress

Majority of the new sites and finished rips are made using the Genesis Framework. In Wordpress, we extend a framework using Child Themes.

↓

So Rule #1...

Never modify framework files

  • Not any single file of the framework should be edited
  • Our workplace for this scenario is the child theme itself
  • We modify the framework through the child theme

Some useful snippets for Genesis:


<?php
/** Remove Title & Description */
remove_action( 'genesis_site_title', 'genesis_seo_site_title' );
remove_action( 'genesis_site_description', 'genesis_seo_site_description' );

/** Remove post titles */
remove_action( 'genesis_post_title','genesis_do_post_title' );

/** Remove Edit Link */
add_filter( 'edit_post_link', '__return_false' );

/** Add custom sidebar */
genesis_register_sidebar(array(
    'name'=>'Alternative Sidebar',
    'id' => 'sidebar-alternative',
));

Some useful snippets for Genesis:


<?php
/** Remove footer widgets */
remove_theme_support( 'genesis-footer-widgets', 3 );
